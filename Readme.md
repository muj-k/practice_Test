オブジェクト指向と例外処理を意識して書きました


■ コマンド仕様と出力要件
各コマンドを処理するたびに、必ず1行の実行結果を標準出力（print）してください。
複数のエラー条件に該当する場合は、記載されている順番が早いエラーを優先して出力してください。

1. ユーザー登録
ADD_USER [ユーザーID] [ユーザー名]

処理: 新規ユーザーをシステムに登録します。

出力:

成功時: SUCCESS

エラー: すでに同じ ユーザーID が存在する場合、ERR_USER_EXISTS

2. 商品の出品
ADD_ITEM [商品ID] [商品名] [初期価格] [出品者ユーザーID]

処理: 指定された初期価格で商品をシステムに登録し、オークションを開始（出品中状態に）します。

出力:

成功時: SUCCESS

エラー1: すでに同じ 商品ID が存在する場合、ERR_ITEM_EXISTS

エラー2: 出品者ユーザーID がシステムに存在しない場合、ERR_USER_NOT_FOUND

3. 入札
BID [商品ID] [入札者ユーザーID] [入札額]

処理: 指定された商品に入札します。

出力:

成功時: SUCCESS

エラー1: 商品ID が存在しない場合、ERR_ITEM_NOT_FOUND

エラー2: 入札者ユーザーID が存在しない場合、ERR_USER_NOT_FOUND

エラー3: その商品のオークションが既に終了している場合、ERR_AUCTION_ENDED

エラー4: 出品者自身が入札しようとした場合、ERR_OWN_ITEM

エラー5: 入札額 が「現在の最高入札額（まだ入札がない場合は初期価格）」以下の場合、ERR_LOW_PRICE

4. オークションの終了
END_ITEM [商品ID]

処理: 商品のオークションを終了し、これ以降の入札を受け付けない状態にします。

出力:

成功時: SUCCESS

エラー1: 商品ID が存在しない場合、ERR_ITEM_NOT_FOUND

エラー2: すでにオークションが終了している場合、ERR_AUCTION_ENDED

5. 商品状態の確認（結果出力）
PRINT_RESULT [商品ID]

処理: 指定された商品の最終結果を出力します。

出力形式: RESULT [商品ID] [商品名] [ステータス] [落札者ID] [最終価格]

[ステータス] は、出品中なら ACTIVE、終了済なら ENDED。

[落札者ID] は、誰も入札していない場合は NONE とする。

成功時出力例: RESULT i01 PC ENDED u02 1500

エラー: 商品ID が存在しない場合、ERR_ITEM_NOT_FOUND
